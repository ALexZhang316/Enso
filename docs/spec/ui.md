# UI Spec

> Opus 负责维护本文件。Codex 负责实现并反馈可行性。

## 范围

定义 Enso 桌面 UI 的交互原则、布局契约、状态展示规则、确认时机。

## 设计原则

- 简洁优先
- 主聊天优先
- 固定三栏布局
- 不自动路由模式
- 聚焦于思考、执行和检视
- 个人执行工作台，不是消费级聊天应用

## 总体布局

固定三列：

| 区域 | 宽度建议 | 职责 |
|------|---------|------|
| 左栏 (left rail) | 20% | 模式切换 / 会话列表 / 全局导航 |
| 中央面板 (center pane) | 55% | 主聊天 + 输入 + 知识/设置/审计视图 |
| 右栏 (right rail) | 25% | 上下文 / 计划 / 轨迹 / 验证 / 证据 / 状态 / 审计 |

左右栏可折叠，但默认展开。

## 左栏

### 模式选择器

三个切换按钮：Deep Dialogue / Decision / Research

规则：
- 点击未激活的按钮 -> 激活该模式（互斥）
- 点击已激活的按钮 -> 取消激活，回到 Default 模式
- Default 模式时三个按钮均显示为未激活
- 不自动检测
- 切换时不弹出新窗口

### 会话列表

功能：
- 创建会话
- 切换会话
- 重命名（双击）
- 删除（带确认）
- 置顶/取消置顶

排序：置顶优先，然后按更新时间降序。

### 全局导航

底部按钮：
- 知识库 -> 切换中央面板到知识视图
- 设置 -> 切换中央面板到设置视图
- 审计记录 -> 切换中央面板到审计视图

点击当前已激活的导航按钮 -> 回到聊天视图（toggle 行为）。

## 中央面板

### 聊天头部

显示：
- 会话标题
- 当前模式标签（Default 模式时隐藏）
- 是否已附加知识来源
- 版本徽章

### 消息流

- 用户消息右对齐，主色背景
- 助手消息左对齐，白色背景带阴影
- 系统消息左对齐，橙色背景
- 支持 Markdown 渲染（标题/列表/代码块/引用/粗体）
- 助手消息底部显示元信息：检索数量、工具名称、工具摘要

### 输入区

- 多行文本框
- 发送按钮（提交时禁用空输入）
- 知识导入按钮
- 检索开关（本轮启用检索）
- 状态行：显示分类结果、错误信息、导入状态

### 知识视图

- 导入按钮
- 来源列表（名称、路径、分块数）
- 导入状态提示

### 设置视图

可配置项：
- 提供商选择（下拉菜单，包含所有预设提供商）
- 模型选择（随提供商变化）
- 基础 URL 选择（随提供商变化）
- API Key 输入（密码字段，存入安全存储，不写入 TOML）
- 表达偏好：密度（精简/标准/详尽）、结构化优先
- 汇报粒度（方案级/结果级）
- 权限配置：四种 action type x 三级 permission
- 模式默认值：按模式启用检索

### 审计视图

- 按会话过滤开关
- 刷新按钮
- 审计记录列表：模式、结果类型、会话 ID、检索/工具使用、风险、时间

## 右栏

### 上下文卡片

显示：模式名称和描述、提供商、模型、知识来源数、工作区根、密度、汇报粒度。

### 计划面板

显示：目标、步骤列表、工具、验证目标。无计划时显示"无计划 (纯对话)"。

### 轨迹面板

显示：每个 trace entry 的阶段标签 + 时间 + 摘要。无轨迹时显示"无轨迹"。

### 验证面板

显示：验证状态（通过/跳过/被拦截/失败）+ 详细信息。按状态着色。

### 证据面板

显示：最近一次助手回复的检索片段（最多 3 个）。

### 待确认动作面板

显示：pendingAction 的摘要、目标路径或命令。无时显示"无待确认动作"。

### 状态面板

显示：检索是否使用、工具列表、是否待确认、任务状态。
待确认时显示确认按钮，按钮文案随 action kind 变化。

### 审计摘要面板

显示：模式、检索、工具、类型、风险。

## 确认交互

- 确认按钮仅在 pendingConfirmation = true 时显示
- 点击确认 -> 调用 resolvePendingConfirmation
- 加载中或提交中时按钮禁用
- 确认按钮文案：
  - workspace_write -> "确认并执行工作区写入"
  - host_exec -> "确认并执行工作区命令"
  - 其他 -> "确认并清除门控"

## 启动状态

- 打开最后一个会话或新建空会话
- 模式 = Default
- 输入框立即可用
- 右栏显示当前上下文和配置摘要
- 无引导页、模板页或推荐页

## 显式延后的 UI 功能

- 面板自由拖拽
- 多窗口工作区
- 画布/白板
- 分支树
- 浮动卡片
- 标签/项目管理
- 自动弹出推荐面板
- 丰富动画
- 社交频道收件箱

## 验收标准

1. 三栏布局必须在 960px 以上宽度正常显示
2. 模式切换必须互斥，且可回到 Default
3. 会话 CRUD 必须正常工作
4. 聊天消息必须正确渲染 Markdown
5. 右栏面板必须实时反映当前状态
6. 确认按钮必须仅在 pending 时显示
7. 设置修改必须持久化到 TOML
8. API Key 必须通过安全存储，不出现在 TOML 或 SQLite 明文中
