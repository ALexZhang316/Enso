# Baseline v0.3.4

## 产品身份

Enso 是单用户、本地优先的 Windows 桌面个人代理。

核心不是社交存在、独立人格或频道覆盖，而是：
- 高控制本地执行
- 一等工具使用
- 有界高权限操作
- 复杂问题规划
- 可见验证与审计
- 深度用户定制

主框架：
- 主聊天窗口作为控制面
- UI 背后的执行内核
- planner -> executor -> verifier 主循环
- 工作区、状态、审计作为一等系统

## 本地优先的含义

本地优先不意味着所有推理都在本地运行。

当前阶段默认本地的内容：
- 配置和策略文件
- 会话状态和会话状态
- 审计和执行日志
- 工作区文件和输出
- 知识索引和背景文档
- 权限门控和工作流路由决策

允许云调用的场景：
- 模型推理
- Web 检索
- 显式外部能力

云组件是附加能力，不是产品的治理大脑。

## 模式基线

Enso 始终运行在一个活跃模式下。四种可能状态：
- Default（无需用户选择）
- Deep Dialogue（手动选择，偏重连续性和概念）
- Decision（手动选择，偏重决策支持）
- Research（手动选择，偏重证据和文档）

规则：
- 三个可选模式互斥
- 不自动路由模式
- 模式偏置共享执行核心，不替换它

模式影响：工具使用倾向、推理姿态、记忆读写倾向、澄清阈值、输出形态。

## 功能需求

- FR-1 主聊天控制面
- FR-2 本地数据所有权
- FR-3 模式特定推理姿态
- FR-4 知识访问
- FR-5 一等工具访问
- FR-6 权限门控
- FR-7 可见追溯性
- FR-8 结果验证

## 当前系统范围

在范围内：
- 稳定三区域主窗口
- 本地配置、日志、工作区和状态持久化
- 基本知识附加和检索
- 工具注册：read / search / compute / workspace-write / exec
- 写入或副作用操作的审批门控
- 可见计划和执行轨迹
- 可读审计摘要
- 基本结果验证

不在范围内：
- 连接器市场
- 社交消息集成
- 伴侣人格界面
- 可见多代理编排
- 重度游戏化 / 社交界面
- 自动长期人格记忆
- 当前阶段的常驻守护自主性

## 不变量

- Windows 桌面主窗口形态
- Default 模式 + 手动可选模式
- 本地优先控制平面
- 单信任操作员边界
- 可见权限门控
- 可审计动作和结果
- 可配置行为和策略

## 去强调的方向

不以以下为产品组织中心：
- 伴侣人格目标
- 社交或消息应用入口
- 独立常开生活助手框架
- 广泛频道扩展
- 氛围优先界面决策

## OpenClaw 经验

保留的架构经验：
- 类型化一等工具
- 工作区中心执行
- 严格配置和策略验证
- 单信任操作员边界
- 显式审批 / 允许列表思维
- 本地控制平面思维

不复制：
- 多频道助手产品形态
- 全覆盖产品价值
- 伴侣身份
- 社交接管策略

## 能力目标

每次实质迭代应改进以下一项或多项：
- 工具可靠性
- 规划质量
- 验证质量
- 工作区产物生成
- 状态持久化和可恢复性
- 权限边界清晰度
- 用户定制工作流适配

当前实现优先级：
- 使单次请求链显式可检视
- 将检索和类型化工具连接到主执行路径
- 存储和渲染计划 / 执行轨迹 / 验证作为一等状态
- 保持高权限操作作为 proposal 门控，直到安全执行边界就绪

## 成功标准

Enso 应在以下方面逐步优于通用聊天界面：
- 真实本地计算机任务
- 文档支持的研究任务
- 结构化决策支持
- 带验证的多步骤问题解决
