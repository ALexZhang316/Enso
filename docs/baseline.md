# Baseline v2.0.0

## 产品身份

Enso 是单用户、本地优先的 Windows 桌面个人 AI 助手。

核心定位：能力优先的三板块对话工具。不是企业级代理框架，不是通用执行引擎。

设计原则：
- 直接、高质量的对话体验
- 三个板块各司其职，互不干扰
- 多提供商接入，用户自主选择
- 本地数据所有权
- 最小复杂度

## 三大板块

Enso 的全部功能围绕三个严格隔离的板块展开：

### 深度对话 (dialogue)
哲学、艺术、宗教、美学等长文字探讨。不需要工具，纯文本对话。高温度、长历史窗口，追求思维连续性和概念深度。

### 投资决策 (decision)
美股及衍生品的长期资产配置。需要网络搜索获取最新信息。中等温度，偏重事实和数据支持。

### 科研辅助 (research)
临床医学文献检索、实验数据处理、论文写作辅助。需要学术搜索工具。低温度，偏重证据密度和学术规范。

规则：
- 三个板块互斥，用户手动切换
- 不自动路由、不自动切换
- 没有"默认模式"
- 每个板块独立的会话列表和系统 prompt

## 本地优先的含义

本地存储的内容：
- 所有会话和消息（SQLite）
- 提供商配置（TOML）
- API Key（Electron 加密存储）
- 应用状态

需要云调用的场景：
- 模型推理（四家提供商 API）
- Web 搜索（待实现）
- 学术检索（待实现）

## 不变量

- Windows 桌面主窗口形态
- 三板块互斥，用户选择
- 本地优先数据所有权
- 单信任操作员边界
- 能力优先，不是防御优先

## 不在范围内

- 自动模式切换
- Default / 日常模式
- 权限门控和审批流程
- 审计系统
- 知识库 / RAG
- 执行流水线（规划 → 执行 → 验证）
- Office / Windows 系统自动化
- 连接器市场
- 社交消息集成
- 自动长期人格记忆
- 常驻守护自主性

## 能力目标

每次迭代应改进以下一项或多项：
- 对话质量和连续性
- 工具可靠性（搜索、检索）
- 多提供商体验一致性
- UI 交互流畅度

当前优先级：
- 完善三板块对话体验
- 实现 decision 板块的网络搜索工具
- 实现 research 板块的学术检索工具
- 投资组合跟踪表

## 成功标准

Enso 应在以下方面优于直接使用各 AI 网页版：
- 深度对话的连续性和 prompt 定制
- 投资信息的聚合和分析
- 学术文献的系统化检索和写作辅助
- 跨提供商的统一体验
